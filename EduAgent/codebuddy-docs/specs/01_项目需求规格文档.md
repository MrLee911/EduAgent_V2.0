## 2. 项目背景

在高校、职业教育和在线教学场景中，教师通常会积累大量课程资料，例如：

```text
课件
教材
讲义
实验指导书
作业说明
课程大纲
教学案例
参考代码
```

但这些资料存在几个常见问题：

1. 学生查找资料困难。
2. 学生提问重复，教师答疑压力大。
3. 教师出题和设计任务耗时。
4. 教师难以快速分析学生关注的知识点。
5. 教学报告和学情分析主要依赖人工整理。
6. 课程资源质量缺少自动分析。
7. 学生缺少基于课程内容的个性化学习路径。
8. 课程 AI 助手往往只是普通聊天机器人，缺少工具调用、技能复用和业务落库能力。

EduAgent 的目标是将课程资料、教学业务、RAG、MCP、Skills 和多智能体工作流结合起来，构建一个能真正服务教学过程的课程智能体平台。

------

## 3. 项目一句话说明

EduAgent 是一个面向教学场景的课程智能体平台，支持教师上传课程资料、构建课程知识库、生成教学任务和教学报告，并通过 MCP 工具生态与 Skills 技能系统扩展课程资源分析、代码辅导、教学设计和学习路径推荐等智能体能力。

## 4. 项目建设目标

### 4.1 基础业务目标

系统应完成以下基础业务闭环：

```text
教师创建课程
→ 学生加入课程
→ 教师上传课程资料
→ 系统构建课程知识库
→ 学生基于课程资料问答
→ 教师生成教学任务
→ 教师发布任务
→ 学生查看任务
→ 教师生成教学报告
```

------

### 4.2 智能体平台目标

系统还应完成以下智能体能力闭环：

```text
课程资料上传
→ ResourceAnalysisAgent 分析资源
→ Skills 提取知识点和资料缺口
→ CourseQAAgent 支持课程问答
→ TaskGenerationAgent 生成任务
→ LessonDesignAgent 生成教学设计
→ ReportGenerationAgent 生成教学报告
→ StudyPathAgent 推荐学习路径
→ MCP Tool 调用真实工具
→ Agent Runs 记录执行轨迹
```

------

### 4.3 教学价值目标

系统应帮助教师：

1. 降低重复答疑成本。
2. 提高课程资源利用率。
3. 快速生成教学任务。
4. 快速生成教学报告。
5. 分析课程资料缺口。
6. 分析学生高频问题。
7. 辅助教学设计和备课。
8. 基于真实数据改进教学。

系统应帮助学生：

1. 更方便地理解课程资料。
2. 获得基于课程知识库的问答支持。
3. 获得引导式作业帮助，而不是直接答案。
4. 获得代码解释和学习建议。
5. 获得个性化学习路径。

系统应帮助管理员：

1. 管理用户。
2. 管理系统配置。
3. 管理 Skills 能力。
4. 管理 MCP 工具。
5. 审计 Agent / Skill / MCP 执行记录。

## 5. 用户角色

系统包含三类系统角色：

```text
student
teacher
admin
```

课程内还存在课程成员角色：

```text
student
teacher
```

------

### 5.1 学生 student

学生主要使用系统进行课程学习。

学生可以：

1. 注册和登录。
2. 查看个人信息。
3. 加入课程。
4. 查看已加入课程。
5. 查看课程资料列表。
6. 基于课程资料进行智能问答。
7. 查看自己的问答历史。
8. 对 AI 回答点赞或点踩。
9. 查看教师发布的教学任务。
10. 查看任务详情。
11. 获取学习路径建议。
12. 使用受限的代码解释能力。

学生不可以：

1. 创建课程。
2. 上传课程资源。
3. 删除课程资源。
4. 生成教学任务。
5. 发布教学任务。
6. 生成教学报告。
7. 查看教学报告。
8. 管理课程成员。
9. 管理 Skills。
10. 管理 MCP。
11. 查看 Agent 全局执行记录。
12. 访问其他课程数据。
13. 获取完整可提交作业答案。

------

### 5.2 教师 teacher

教师主要使用系统进行课程建设和教学管理。

教师可以：

1. 注册和登录。
2. 创建课程。
3. 编辑自己创建的课程。
4. 删除自己创建的课程。
5. 管理课程成员。
6. 上传课程资源。
7. 批量上传课程资源。
8. 查看资源处理状态。
9. 重新处理课程资源。
10. 删除课程资源。
11. 使用课程知识库问答。
12. 生成课堂练习。
13. 生成课后作业。
14. 生成实验指导。
15. 发布、归档、删除教学任务。
16. 生成周报、月报、学期报告。
17. 查看教学报告。
18. 导出 Markdown 报告。
19. 分析课程资源。
20. 生成教学设计。
21. 生成测验题。
22. 使用代码解释和代码分析能力。
23. 查看自己课程范围内的 Agent / Skill / MCP 执行记录。

教师不可以：

1. 管理全局用户。
2. 管理全局系统设置。
3. 管理全局 MCP Server。
4. 查看其他教师课程数据。
5. 查看其他课程的学生个人学习记录。

------

### 5.3 管理员 admin

管理员负责平台管理和运维审计。

管理员可以：

1. 登录系统。
2. 查看用户列表。
3. 创建用户。
4. 启用或禁用用户。
5. 修改用户角色。
6. 查看系统设置。
7. 查看 Skills 列表。
8. 管理 Skills 启用状态。
9. 查看 MCP Server 列表。
10. 管理 MCP Server 配置。
11. 查看 MCP Tool 列表。
12. 查看 MCP Tool 调用记录。
13. 查看 Agent 执行记录。
14. 查看 Skill 执行记录。
15. 辅助排查系统问题。

管理员不应直接：

1. 查看用户密码。
2. 查看真实 API Key。
3. 查看完整 JWT Secret。
4. 绕过日志直接操作用户隐私数据。
5. 让工具执行原始 SQL。
6. 让 MCP Tool 跨越权限读取敏感数据。

## 8. 核心业务流程

### 8.1 用户注册与登录流程

```text
用户打开系统
→ 选择注册或登录
→ 输入账号信息
→ 后端校验用户
→ 生成 access_token 和 refresh_token
→ 前端保存用户信息和 Token
→ 跳转首页
```

需求要求：

1. 学生和教师可通过注册页创建账号。
2. 管理员账号应由系统预置或管理员创建。
3. 登录成功应返回 access_token 和 refresh_token。
4. 前端应支持 Token 自动刷新。
5. Token 失效且刷新失败时，应自动退出登录。

------

### 8.2 课程创建与加入流程

教师流程：

```text
教师登录
→ 创建课程
→ 系统生成课程码
→ 教师分享课程码给学生
```

学生流程：

```text
学生登录
→ 输入课程码
→ 加入课程
→ 查看课程资料和任务
```

需求要求：

1. 课程码唯一。
2. 学生不能重复加入同一课程。
3. 课程成员关系必须保存。
4. 课程数据必须通过 `course_id` 隔离。

------

### 8.3 课程资源处理流程

```text
教师上传资源
→ 后端保存文件
→ 创建 Resource 记录
→ 投递 Celery 任务
→ 文档解析
→ 文本切片
→ Embedding
→ 写入 ChromaDB
→ 写入 chunks 表
→ Resource 状态变为 ready
```

需求要求：

1. 上传文件必须校验类型和大小。
2. 支持单文件上传和批量上传。
3. 资源处理为异步任务。
4. 前端可轮询资源处理状态。
5. 处理失败必须记录错误信息。
6. 删除资源时应清理文件、chunks 和向量数据。

------

### 8.4 课程问答流程

```text
学生提出问题
→ 输入护栏检查
→ 查询改写
→ 检索课程知识库
→ Reranker 精排
→ 构造 Prompt
→ 调用 LLM
→ 输出护栏检查
→ 保存 qa_records
→ 返回答案和 sources
```

需求要求：

1. 问答必须限定当前课程。
2. 回答必须优先基于课程资料。
3. 资料不足时必须明确说明。
4. 回答必须返回引用来源。
5. 学生作业类问题不得直接给完整答案。
6. 流式问答必须支持 SSE。
7. 多轮对话应支持 `conversation_id`。

------

### 8.5 教学任务生成流程

```text
教师输入主题、类型、难度、额外要求
→ 检索课程资料
→ 调用 task_generation Skill
→ 生成 Markdown 任务
→ 保存为 draft
→ 教师编辑
→ 教师发布
→ 学生查看
```

需求要求：

1. 只有教师或管理员可生成任务。
2. 任务生成必须基于当前课程资料。
3. 默认生成草稿。
4. 学生只能查看已发布任务。
5. 支持课堂练习、课后作业、实验指导。
6. 后续应支持独立测验题生成 Skill。

------

### 8.6 教学报告生成流程

```text
教师选择报告类型和日期范围
→ 查询课程统计
→ 查询问答热点
→ 查询资源统计
→ 查询任务统计
→ 调用 report_generation Skill
→ 生成 Markdown 报告
→ 保存 reports
→ 教师查看和导出
```

需求要求：

1. 只有教师或管理员可生成报告。
2. 报告必须基于真实统计数据。
3. 不得编造学生人数、资源数、任务数、问答数。
4. 报告建议必须具体可执行。
5. Markdown 导出必须稳定。
6. PDF 导出可作为后续增强。

------

### 8.7 资源分析流程

```text
教师选择课程资源
→ 调用 resource_analysis Skill
→ File Resource MCP Tool 读取资源文本
→ LLM 提取摘要和知识点
→ 输出资源摘要、知识点、缺失内容和教学建议
```

需求要求：

1. 只有教师或管理员可分析课程资源。
2. 分析必须基于真实资源内容。
3. 不得编造文件内容。
4. 分析结果可用于教学设计、任务生成和报告生成。
5. 后续可保存资源分析结果。

------

### 8.8 教学设计流程

```text
教师输入教学主题、课时、学生水平和教学目标
→ 检索课程资料
→ 调用 lesson_design Skill
→ 生成教学设计 Markdown
```

需求要求：

1. 只有教师或管理员可生成教学设计。
2. 教学设计应包含教学目标、重点、难点、流程、活动、练习和评价。
3. 课时分配必须合理。
4. 内容应基于课程资料和教师目标。

------

### 8.9 学习路径推荐流程

```text
学生选择目标知识点
→ 查询该学生历史问答
→ 查询课程资源和知识点
→ 调用 study_path Skill
→ 返回学习路径、推荐资源和练习建议
```

需求要求：

1. 学生只能分析自己的学习记录。
2. 不得泄露其他学生数据。
3. 推荐资源必须来自当前课程。
4. 建议应具体、可执行、鼓励性表达。

------

### 8.10 MCP 工具调用流程

```text
Agent / Skill 需要工具
→ Tool Router 选择工具
→ MCP 权限检查
→ MCP Client 调用 Tool
→ 记录 mcp_tool_calls
→ 返回工具结果
→ Skill / Agent 继续执行
```

需求要求：

1. MCP Tool 必须有输入输出 Schema。
2. MCP Tool 必须有权限检查。
3. MCP Tool 不得执行原始 SQL。
4. MCP Tool 不得返回真实服务器路径。
5. MCP Tool 不得跨课程访问数据。
6. MCP Tool 调用必须可审计。

# M01 用户认证与权限

## M01.1 用户注册

用户故事：

```text
作为学生或教师，我希望可以注册账号，以便进入系统使用课程功能。
```

需求：

1. 支持用户名、邮箱、密码注册。
2. 普通注册允许选择学生或教师。
3. 不允许普通用户注册管理员账号。
4. 用户名唯一。
5. 邮箱唯一。
6. 密码必须加密存储。
7. 注册成功返回用户基础信息。

验收标准：

1. 学生注册成功。
2. 教师注册成功。
3. 重复用户名注册失败。
4. 重复邮箱注册失败。
5. 密码不以明文保存。

------

## M01.2 用户登录与 Token 刷新

需求：

1. 支持用户名或邮箱登录。
2. 登录成功返回 access_token。
3. 登录成功返回 refresh_token。
4. access_token 过期后前端可用 refresh_token 刷新。
5. refresh_token 无效时要求重新登录。
6. 用户禁用后不能继续访问系统。

验收标准：

1. 登录成功后前端进入首页。
2. 401 后自动刷新 Token。
3. 刷新失败自动退出登录。
4. 禁用用户无法继续访问。

------

## M01.3 角色权限

需求：

1. 学生只能访问学生权限范围。
2. 教师只能管理自己的课程。
3. 管理员可以访问管理后台。
4. 后端必须进行最终权限校验。
5. 前端权限只作为体验层限制。

验收标准：

1. 学生访问教师接口返回 403。
2. 非课程成员访问课程接口返回 403。
3. 非管理员访问 `/admin/*` 返回 403。

6. Skill 必须有输入输出 Schema。
7. MCP Tool 必须有输入输出 Schema。
8. 权限必须由代码判断，不能由 LLM 判断。
9. 所有课程内查询必须限定 course_id。
10. 所有 AI 输出必须经过必要护栏。
11. 新增数据库表必须有 Alembic 迁移。
12. 新增前端页面必须补充路由和权限。
13. 新增 API 必须同步 API 文档。
14. 新增 Prompt 必须同步提示词文档。
15. 新增 Skill 必须有 `SKILL.md`。
16. 新增 MCP Tool 必须有权限配置和测试。
17. 敏感信息不得写入代码和文档。
18. Mock LLM 节点不得进入生产主链路。

## 16. 本文档总结

新版 EduAgent 的需求范围已经从：

```text
课程管理 + RAG 问答 + 任务生成 + 报告生成
```

升级为：

```text
课程智能体平台
= 课程业务系统
+ RAG 知识库
+ MCP 工具生态
+ Skills 技能系统
+ Agent Orchestrator
+ Planner
+ Tool Router
+ Skill Router
+ 多智能体协作
+ 执行审计
```

当前优先级是：

```text
1. 先修复现有 P0 问题
2. 稳定用户、课程、资源、问答、任务、报告主链路
3. 新增 Skills 基础框架
4. 新增 MCP 基础框架
5. 新增 Agent Orchestrator / Planner / Router
6. 扩展资源分析、代码辅导、教学设计、学习路径等智能体
7. 补充前端页面、测试、审计和部署能力
```

最终目标是让 EduAgent 成为一个可演示、可教学、可继续扩展的课程智能体平台，而不是停留在普通 RAG 聊天机器人。
