# 05 AI 智能体行为定义

> 项目名称：EduAgent 课程资源与教学任务智能体
> 文档类型：AI 智能体架构、MCP 工具生态、Skills 技能系统与工作流设计文档
> 适用对象：CodeBuddy / AI 编程助手 / Agent 开发 / RAG 开发 / MCP 开发 / 后端开发 / 测试人员
> 对应代码：`backend/app/agent/`、`backend/app/rag/`、`backend/app/services/`、`backend/app/tasks/`、`backend/app/mcp/`、`backend/app/skills/`
> 文档版本：v2.0
> 优化日期：2026-06-10

------

## 1. 文档目的

本文档用于重新设计 EduAgent 项目的 AI 智能体系统。

原有智能体设计主要围绕：

```text
RAG 检索
→ LLM 回答
→ 任务生成
→ 报告生成
```

这种设计更接近“带知识库的聊天机器人”，不足以体现真正的智能体能力。

重新设计后的 EduAgent 智能体应具备：

1. **课程知识理解能力**：能理解课程资料、资源结构、学生问题、教师教学目标。
2. **RAG 检索增强能力**：能基于课程资料检索、引用、归纳和回答。
3. **MCP 工具调用能力**：能通过 MCP 接入外部工具、数据库、文件系统、知识库、教学平台等能力。
4. **Skills 技能系统能力**：能根据任务类型调用预定义技能，例如课程资料分析、教学任务生成、实验指导生成、报告分析、代码解释等。
5. **任务规划能力**：面对复杂请求时，先拆解任务，再选择工具和技能执行。
6. **多智能体协作能力**：不同 Agent 分工处理问答、资源分析、任务生成、报告生成、代码辅助等任务。
7. **状态管理能力**：能维护 AgentState、工具调用记录、技能调用记录、对话上下文和执行轨迹。
8. **安全护栏能力**：能限制越权访问、跨课程访问、提示词注入、敏感信息泄露和不当输出。
9. **可扩展能力**：后续可以新增 MCP Server、新增 Tool、新增 Skill、新增 Agent，而不破坏现有架构。

本文档重点回答：

1. EduAgent 应设计成什么样的智能体系统。
2. MCP 在本项目中承担什么角色。
3. Skills 在本项目中承担什么角色。
4. Agent 如何选择 Tool 和 Skill。
5. 每类 Agent 的职责是什么。
6. 每个工作流如何编排。
7. 当前已有代码需要如何改造。
8. CodeBuddy 应按什么顺序实现这些能力。

## 2. 重新定位：EduAgent 不是聊天机器人

### 2.1 原有设计问题

原有智能体设计存在以下问题：

1. 主要是用户提问后调用 RAG 回答。
2. 任务生成和报告生成只是简单调用 LLM。
3. 缺少工具生态，无法调用外部服务。
4. 缺少 Skills 技能抽象，无法复用复杂任务能力。
5. 缺少任务规划器，无法对复杂需求进行拆解。
6. Agent 与 Service 层边界不够清晰。
7. LangGraph 工作流中部分节点仍是 Mock。
8. Agent 的能力看起来像“普通聊天 + 知识库问答”。

------

### 2.2 新定位

重新设计后的 EduAgent 应定位为：

```text
面向教学场景的课程智能体平台
```

它不是单一聊天机器人，而是由多个智能体、多个工具、多个技能组成的 AI 教学工作系统。

可以理解为：

```text
EduAgent
= 课程知识库
+ RAG 检索系统
+ MCP 工具生态
+ Skills 技能系统
+ LangGraph 工作流
+ 多智能体协作
+ 教学业务系统
```

------

### 2.3 新架构目标

EduAgent 智能体系统应支持以下能力：

| 能力         | 说明                                                 |
| ------------ | ---------------------------------------------------- |
| 课程问答     | 基于课程资料进行带引用来源的问答                     |
| 资料分析     | 自动分析课程资源结构、摘要、知识点和薄弱环节         |
| 教学任务生成 | 自动生成随堂练习、课后作业、实验指导                 |
| 教学报告生成 | 根据课程数据生成周报、月报、学期报告                 |
| 代码辅助     | 对课程中的代码示例进行解释、纠错、补全和改写         |
| 学习路径建议 | 根据学生历史问答和课程资源推荐学习路径               |
| 教师备课辅助 | 根据课程资料生成教案、课堂活动、知识点讲解           |
| 资源质量评估 | 判断课程资源是否缺失、重复、过难或不完整             |
| MCP 工具调用 | 调用文件、数据库、搜索、代码执行、教学平台等外部工具 |
| Skills 复用  | 将复杂能力封装为可注册、可调用、可测试的技能         |

## 4. Agent 类型设计

重新设计后，EduAgent 应包含多个智能体，而不是一个通用聊天 Agent。

| Agent                   | 中文名称           | 主要职责                                 |
| ----------------------- | ------------------ | ---------------------------------------- |
| `CourseQAAgent`         | 课程问答智能体     | 基于课程资料回答学生问题                 |
| `ResourceAnalysisAgent` | 课程资源分析智能体 | 分析课程资料结构、摘要、知识点、缺失内容 |
| `TaskGenerationAgent`   | 教学任务生成智能体 | 生成随堂练习、课后作业、实验指导         |
| `ReportGenerationAgent` | 教学报告生成智能体 | 生成周报、月报、学期报告                 |
| `CodeTutorAgent`        | 代码辅导智能体     | 解释课程代码、纠错、生成示例             |
| `LessonDesignAgent`     | 教学设计智能体     | 生成教案、课堂活动、教学目标             |
| `StudyPathAgent`        | 学习路径智能体     | 根据学生问题和课程资料推荐学习路径       |
| `AdminAnalysisAgent`    | 管理分析智能体     | 辅助管理员分析系统数据和用户活跃情况     |
| `MCPToolAgent`          | MCP 工具智能体     | 负责发现、选择和调用 MCP 工具            |
| `SkillExecutionAgent`   | 技能执行智能体     | 负责选择和执行 Skills                    |

## 5. MCP 功能设计

## 5.1 MCP 在 EduAgent 中的定位

MCP 用于将 EduAgent 与外部工具、外部数据源、外部服务连接起来。

在 EduAgent 中，MCP 不只是“工具调用”，而是智能体能力扩展层。

MCP 可以接入：

1. 文件系统。
2. 数据库。
3. 课程平台。
4. 教学资源库。
5. 代码执行环境。
6. 搜索服务。
7. 表格分析工具。
8. 文档解析工具。
9. 外部知识库。
10. 学习管理系统。

------

## 5.3 MCP Client 设计

推荐新增：

```text
backend/app/mcp/client.py
```

职责：

1. 连接 MCP Server。
2. 获取可用工具列表。
3. 获取工具 Schema。
4. 执行工具调用。
5. 处理工具调用结果。
6. 记录工具调用日志。
7. 处理超时和异常。
8. 做权限检查。

核心接口建议：

```python
class MCPClient:
    async def list_tools(self, server_name: str) -> list[MCPToolDefinition]:
        ...

    async def call_tool(
        self,
        server_name: str,
        tool_name: str,
        arguments: dict,
        context: MCPExecutionContext,
    ) -> MCPToolResult:
        ...

    async def health_check(self, server_name: str) -> bool:
        ...
```

## 5.4 MCP Server Registry 设计

推荐新增：

```text
backend/app/mcp/registry.py
```

职责：

1. 注册 MCP Server。
2. 管理 Server 名称、地址、权限、启用状态。
3. 提供可用工具查询。
4. 支持按课程、角色、环境启用工具。

示例配置：

```python
MCP_SERVERS = {
    "course_db": {
        "name": "course_db",
        "transport": "stdio",
        "enabled": True,
        "description": "课程数据库查询工具",
        "allowed_roles": ["teacher", "admin"],
    },
    "rag_search": {
        "name": "rag_search",
        "transport": "internal",
        "enabled": True,
        "description": "课程知识库检索工具",
        "allowed_roles": ["student", "teacher", "admin"],
    },
    "code_sandbox": {
        "name": "code_sandbox",
        "transport": "http",
        "enabled": True,

        "allowed_roles": ["teacher", "admin"],
    },
}
```

## 5.5 推荐内置 MCP Server

### 5.5.1 Course DB MCP Server

用途：

让智能体通过安全接口查询课程业务数据。

工具示例：

| 工具                  | 说明             |
| --------------------- | ---------------- |
| `get_course_info`     | 获取课程基本信息 |
| `list_course_members` | 获取课程成员     |
| `get_course_stats`    | 获取课程统计     |
| `list_tasks`          | 查询课程任务     |
| `list_reports`        | 查询课程报告     |
| `query_qa_records`    | 查询问答统计     |

权限：

1. 学生只能查询自己课程内允许公开的数据。
2. 教师可以查询自己课程的教学数据。
3. 管理员可以查询全局数据。
4. 不允许原始 SQL 直连数据库。

------

### 5.5.2 File Resource MCP Server

用途：

让智能体访问和分析课程资源文件。

工具示例：

| 工具                         | 说明             |
| ---------------------------- | ---------------- |
| `list_course_files`          | 列出课程文件     |
| `get_file_metadata`          | 获取文件元信息   |
| `extract_file_text`          | 抽取文件文本     |
| `summarize_file`             | 生成文件摘要     |
| `compare_files`              | 对比两个资源文件 |
| `detect_duplicate_resources` | 检测重复资源     |

权限：

1. 只能访问当前课程资源。
2. 学生不能读取教师未公开的资源。
3. 不返回本地真实文件路径。
4. 不允许路径穿越。

------

### 5.5.3 RAG Search MCP Server

用途：

将课程知识库检索能力封装为 MCP 工具。

工具示例：

| 工具                      | 说明                           |
| ------------------------- | ------------------------------ |
| `search_course_knowledge` | 检索课程知识库                 |
| `search_resource_chunks`  | 检索指定资源切片               |
| `get_chunk_detail`        | 获取切片详情                   |
| `get_related_chunks`      | 获取相关切片                   |
| `rebuild_resource_index`  | 重建资源索引，教师或管理员权限 |

权限：

1. 所有检索必须限定 `course_id`。
2. 学生只能检索自己加入课程的数据。
3. 重建索引只允许教师或管理员。

------

### 5.5.4 Code Sandbox MCP Server

用途：

用于代码教学场景，例如代码解释、代码运行、代码纠错、生成测试用例。

工具示例：

| 工具                    | 说明             |
| ----------------------- | ---------------- |
| `run_python_code`       | 运行 Python 代码 |
| `analyze_code_error`    | 分析代码错误     |
| `generate_test_cases`   | 生成测试用例     |
| `explain_code`          | 解释代码         |
| `check_assignment_code` | 检查学生代码结构 |

权限：

1. 默认只允许教师使用代码运行。
2. 学生代码运行需限制时间、内存、网络和文件系统。
3. 禁止执行危险系统命令。
4. 禁止访问宿主机文件。
5. 必须记录执行日志。

## 5.6 MCP 权限模型

每次 MCP 工具调用必须带上下文：

```json
{
  "user_id": "uuid",
  "user_role": "teacher",
  "course_id": "uuid",
  "course_role": "teacher",
  "request_id": "uuid",
  "conversation_id": "uuid"
}
```

权限检查规则：

| 工具类型       | 学生               | 教师               | 管理员 |
| -------------- | ------------------ | ------------------ | ------ |
| 课程知识库检索 | ✅ 当前课程         | ✅ 当前课程         | ✅      |
| 课程资源列表   | ✅ 当前课程公开资源 | ✅ 当前课程全部资源 | ✅      |
| 课程统计       | ❌ 或有限           | ✅ 当前课程         | ✅      |
| 用户管理       | ❌                  | ❌                  | ✅      |
| 资源重建索引   | ❌                  | ✅ 当前课程         | ✅      |
| 代码沙箱       | 受限               | ✅                  | ✅      |
| 文件元数据     | ✅ 当前课程         | ✅ 当前课程         | ✅      |
| 系统设置       | ❌                  | ❌                  | ✅      |

## 6. Skills 功能设计

## 6.1 Skills 在 EduAgent 中的定位

Skills 是 EduAgent 的“可复用能力单元”。

MCP 偏向外部工具接入，Skills 偏向业务能力封装。

区别如下：

| 对比项          | MCP Tool                       | Skill                                    |
| --------------- | ------------------------------ | ---------------------------------------- |
| 主要作用        | 调用外部工具或服务             | 封装一类复杂业务能力                     |
| 粒度            | 较小，单个工具函数             | 较大，完整任务流程                       |
| 示例            | 搜索知识库、读取文件、运行代码 | 生成实验指导、分析课程资源、生成教学报告 |
| 调用方式        | Tool Router 调用               | Skill Router 调用                        |
| 是否可组合      | 可以                           | 可以                                     |
| 是否包含 Prompt | 一般不包含或较少               | 通常包含 Prompt、步骤和输出格式          |
| 是否包含工作流  | 一般没有                       | 可以包含多步骤 workflow                  |

### 6.3.1 SKILL.md

用于描述技能用途、输入、输出、限制和示例。

模板：

```markdown

## 1. 技能定位

说明该技能解决什么问题。

## 2. 适用场景

列出适用的用户请求类型。

## 3. 输入参数

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|

## 4. 输出结果

说明返回结构。

## 5. 可调用工具

列出该技能可以调用的 MCP Tool 或 Local Tool。

## 6. 执行步骤

1. 第一步
```

## 11. 安全护栏重新设计

## 11.1 输入护栏

输入护栏检查：

1. 越狱请求。
2. Prompt Injection。
3. 索要系统 Prompt。
4. 索要 API Key。
5. 跨课程访问。
6. 非法工具调用。
7. 非法 MCP 调用。
8. 非法代码执行。
9. 学生索要完整作业答案。
10. 与课程无关请求。

------

## 11.2 Tool 权限护栏

任何 Tool / MCP / Skill 调用前必须校验：

```text
user_id
user_role
course_id
course_role
tool_name
skill_name
operation_type
```

拒绝情况：

1. 学生调用报告生成。
2. 学生调用课程成员列表。
3. 学生调用资源重建索引。
4. 学生调用管理员工具。
5. 任意用户访问非当前课程资源。
6. 任意用户传入原始 SQL。
7. 任意用户执行危险代码。
8. Agent 试图访问未注册 MCP Server。

------

## 11.3 输出护栏

输出护栏检查：

1. 是否泄露系统 Prompt。
2. 是否泄露密钥。
3. 是否伪造引用来源。
4. 是否跨课程引用。
5. 是否输出完整可提交作业答案。
6. 是否输出危险代码。
7. 是否包含未经授权的学生隐私。
8. 是否把工具内部错误暴露给用户。

## 15. CodeBuddy 实现要求

CodeBuddy 实现 MCP 和 Skills 时必须遵守：

1. 不得删除现有 `services/` 主链路。
2. 不得让 Agent 绕过权限校验。
3. 不得让 MCP Tool 直接执行原始 SQL。
4. 不得让学生调用教师或管理员工具。
5. 不得跨课程访问资源。
6. 不得把 MCP Server 内部错误完整暴露给用户。
7. 不得让 Skill 直接读取 `.env`。
8. 不得让 LLM 自己决定是否有权限，权限必须由代码判断。
9. Skill 输入输出必须有 Schema。
10. MCP Tool 输入输出必须有 Schema。
11. AgentState 必须记录 tool_calls 和 skill_calls。
12. 所有工具调用必须有超时和异常处理。
13. 所有 Skill 必须可单独测试。
14. 所有 MCP Tool 必须可单独测试。
15. 新增 Agent 能力必须同步更新 API 文档、Prompt 文档和测试。

## 17. 本文档总结

重新设计后的 EduAgent 智能体不再是简单聊天机器人，而是：

```text
课程智能体平台
= 多智能体
+ RAG
+ MCP 工具生态
+ Skills 技能系统
+ LangGraph 工作流
+ 权限护栏
+ 教学业务系统
```

核心升级点是：

1. 从“问答机器人”升级为“教学智能体平台”。
2. 从“直接调用 LLM”升级为“Planner + Skill + Tool + LLM”。
3. 从“本地工具函数”升级为“MCP 工具生态”。
4. 从“零散业务逻辑”升级为“可注册、可复用、可测试的 Skills”。
5. 从“单 Agent”升级为“课程问答、资源分析、任务生成、报告生成、代码辅导、教学设计等多 Agent 协作”。
6. 从“结果生成”升级为“可审计、可追踪、可权限控制的智能体执行系统”。

后续 CodeBuddy 应优先完成：

```text
1. 新增 backend/app/skills 基础框架
2. 新增 backend/app/mcp 基础框架
3. 增强 AgentState
4. 新增 Agent Orchestrator / Planner / Router
5. 将 QA / Task / Report 逐步改造成 Skill
6. 将 RAG / Course DB / File / Report Analysis 封装为 MCP Tool
7. 补充权限控制和调用日志
8. 补充测试和文档
```

只有完成 MCP 和 Skills 设计后，EduAgent 才能真正体现“智能体开发”项目的价值，而不是停留在普通 RAG 聊天系统。
