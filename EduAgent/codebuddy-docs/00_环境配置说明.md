# 00 环境配置说明

> 项目名称：EduAgent 课程资源与教学任务智能体
> 文档类型：环境配置、启动部署、模型配置、Agent / MCP / Skills 配置说明
> 适用对象：CodeBuddy / AI 编程助手 / 后端开发 / 前端开发 / Agent 开发 / MCP 开发 / Skills 开发 / 部署人员 / 测试人员
> 对应文件：`.env.example`、`backend/app/config.py`、`backend/requirements.txt`、`frontend/package.json`、`docker-compose.yml`、`docker-compose.prod.yml`、`nginx.conf`

## 1. 文档目的

本文档用于说明 EduAgent 项目的本地开发环境、Docker Compose 环境、后端环境变量、前端环境变量、模型配置、RAG 配置、Agent 配置、MCP 配置、Skills 配置、文件存储配置和部署注意事项。

新版 EduAgent 已经从普通课程 RAG 问答系统升级为：

```text
课程智能体平台
= 课程业务系统
+ RAG 知识库
+ MCP 工具生态
+ Skills 技能系统
+ Agent Orchestrator
+ Planner
+ Tool Router
+ Skill Router
+ 多智能体协作
+ 执行审计
```

因此，环境配置不仅要支持：

```text
FastAPI
Vue 3
PostgreSQL

```

还需要支持：

```text
Agent Orchestrator

Agent Trace

```

## 3. 本地开发环境准备

### 3.1 安装基础软件

本地开发建议安装：

```text
Python 3.11
Node.js 18+
PostgreSQL 15+
Redis 7+
Docker Desktop
Git
```

如果使用 Docker Compose 运行数据库、Redis、ChromaDB 和 MinIO，本机可以不单独安装 PostgreSQL 和 Redis。

------

### 3.2 后端虚拟环境

Linux / macOS：

```bash
cd backend
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Windows PowerShell：

```powershell
cd backend
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
```

------

### 3.3 前端依赖安装

```bash
cd frontend
npm install
```

------

### 3.4 启动基础服务

使用 Docker 启动 PostgreSQL、Redis、ChromaDB、MinIO：

```bash
docker compose up -d postgres redis chromadb minio
```

```bash
docker compose ps
```

## 4. 后端本地启动

### 4.1 准备 `.env`

在项目根目录或后端目录准备 `.env` 文件。

推荐从 `.env.example` 复制：

```bash
cp .env.example .env
```

Windows PowerShell：

```powershell
Copy-Item .env.example .env
```

注意：

```text
.env 不得提交到代码仓库。
.env 中不得写入可以公开传播的真实密钥。
```

------

### 4.2 执行数据库迁移

```bash
cd backend
alembic upgrade head
```

如果使用 `gen_random_uuid()`，迁移中必须启用：

```sql
CREATE EXTENSION IF NOT EXISTS pgcrypto;
```

------

### 4.3 启动后端服务

```bash
cd backend
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

访问：

```text

http://localhost:8000/docs
http://localhost:8000/health
```

### 4.4 启动 Celery Worker


4. ChromaDB 写入。
5. PostgreSQL chunks 写入。



1. 资源解析任务。

5. 资源重建索引任务。

1. 资源分析异步任务。

## 14. LLM 配置

### 14.1 推荐使用 DeepSeek

课程开发中统一推荐使用 DeepSeek，不使用 OpenAI 官方模型作为课程默认方案。

推荐配置：

```env
LLM_PROVIDER=deepseek
LLM_BASE_URL=https://api.deepseek.com/v1
LLM_API_KEY=your_deepseek_api_key
LLM_MODEL_NAME=deepseek-chat
LLM_TEMPERATURE=0.3
LLM_TIMEOUT_SECONDS=120
LLM_MAX_TOKENS=4096
```

说明：

1. DeepSeek API 使用 OpenAI-Compatible 格式。
2. 后端可以通过 `httpx` 调用。
3. `LLM_API_KEY` 必须通过 `.env` 配置。
4. 不得将真实 Key 写入代码和文档。
5. 前端不得直接调用 LLM API。

------

### 14.2 本地开发占位配置

`.env.example` 中可以使用：

```env
LLM_API_KEY=replace_with_your_api_key
```

不要使用真实 Key。

## 15. Embedding 配置

推荐使用 BGE-M3：

```env
EMBEDDING_MODEL_NAME=BAAI/bge-m3
EMBEDDING_DEVICE=cpu
EMBEDDING_NORMALIZE=true
EMBEDDING_BATCH_SIZE=16
EMBEDDING_LOCAL_ONLY=false
EMBEDDING_DIMENSION=1024
```

说明：

1. BGE-M3 向量维度为 1024。
2. 如果更换 Embedding 模型，必须重建 ChromaDB 索引。
3. 如果模型本地缓存不可用，首次加载需要联网下载。
4. 教学场景中可提前下载模型缓存，避免学生运行时报错。
5. 开发环境可以启用 fallback，但生产环境不建议使用 HashEmbeddingFallback。

本地模型缓存建议：

```env
HF_HOME=./models/huggingface
TRANSFORMERS_CACHE=./models/huggingface
```

## 16. Reranker 配置

推荐使用：

```env
RERANKER_ENABLED=true
RERANKER_MODEL_NAME=BAAI/bge-reranker-v2-m3
RERANKER_DEVICE=cpu
RERANKER_TOP_N=5
RERANKER_LOCAL_ONLY=false
```

说明：

1. Reranker 用于精排 RAG 检索结果。
2. Reranker 加载失败时应降级，不应导致问答整体失败。
3. 如果机器资源有限，可以关闭 Reranker：

```env
RERANKER_ENABLED=false
```

## 17. RAG 配置

```env
RAG_CHUNK_SIZE=512
RAG_CHUNK_OVERLAP=64
```

## 21. Agent 配置

新版智能体平台需要新增 Agent 配置。

```env
AGENT_ENABLED=true
AGENT_TRACE_ENABLED=true
AGENT_MAX_STEPS=10
AGENT_DEFAULT_TIMEOUT_SECONDS=120
AGENT_MEMORY_ENABLED=true
AGENT_MEMORY_TTL_SECONDS=86400
AGENT_USE_LANGGRAPH=true
AGENT_MOCK_WORKFLOWS_ALLOWED=false
```

说明：

| 配置                            | 说明                              |
| ------------------------------- | --------------------------------- |
| `AGENT_ENABLED`                 | 是否启用 Agent 编排能力           |
| `AGENT_TRACE_ENABLED`           | 是否记录 Agent 执行轨迹           |
| `AGENT_MAX_STEPS`               | 单次 Agent 最大执行步骤数         |
| `AGENT_DEFAULT_TIMEOUT_SECONDS` | Agent 默认超时时间                |
| `AGENT_MEMORY_ENABLED`          | 是否启用对话记忆                  |
| `AGENT_MEMORY_TTL_SECONDS`      | 对话记忆 TTL                      |
| `AGENT_USE_LANGGRAPH`           | 是否使用 LangGraph 工作流         |
| `AGENT_MOCK_WORKFLOWS_ALLOWED`  | 是否允许 Mock Workflow 进入主链路 |

生产环境建议：

```env
AGENT_ENABLED=true
AGENT_TRACE_ENABLED=true
AGENT_MOCK_WORKFLOWS_ALLOWED=false
```

要求：

1. Mock Workflow 不得进入生产主链路。
2. Agent 执行必须记录 `agent_runs` 和 `agent_steps` 或结构化日志。
3. Agent 不得绕过 Service 层权限。
4. AgentState 必须记录 `skill_calls` 和 `mcp_tool_calls`。

## 23. MCP 配置

```env
MCP_ENABLED=true
MCP_MODE=internal
MCP_TOOL_TIMEOUT_SECONDS=30
MCP_MAX_TOOL_CALLS_PER_RUN=20
MCP_RECORD_TOOL_CALLS=true
MCP_SERVER_CONFIG_PATH=backend/app/mcp/servers.yaml
```

说明：

| 配置                         | 说明                                  |
| ---------------------------- | ------------------------------------- |
| `MCP_ENABLED`                | 是否启用 MCP 工具生态                 |
| `MCP_MODE`                   | MCP 模式：internal / external / mixed |
| `MCP_TOOL_TIMEOUT_SECONDS`   | 单个工具调用超时                      |
| `MCP_MAX_TOOL_CALLS_PER_RUN` | 单次 Agent 最大工具调用次数           |
| `MCP_RECORD_TOOL_CALLS`      | 是否记录 MCP 调用                     |
| `MCP_SERVER_CONFIG_PATH`     | MCP Server 配置文件路径               |

第一阶段建议：

```env
MCP_MODE=internal
```

优先实现内部 MCP Adapter：

```text
internal_rag
```

## 32. 常见问题

### 32.1 登录后自动刷新 Token 失败

检查：

1. `/auth/login` 是否返回 `refresh_token`。
2. 前端是否保存 `refresh_token`。
3. `/auth/refresh` 请求体字段是否为 `refresh_token`。
4. refresh_token 是否过期。
5. 用户是否被禁用。

------

### 32.2 批量上传失败

检查：

1. 前端路径是否为 `/resources/upload-batch`。
2. 后端是否实现 `POST /upload-batch`。
3. FormData 字段是否为 `files`。
4. 文件扩展名是否允许。
5. 文件大小是否超过限制。

------

### 32.3 资源一直处于处理中

检查：

1. Celery Worker 是否启动。
2. Redis 是否可连接。
3. 文件是否解析失败。
4. Embedding 模型是否加载成功。
5. ChromaDB 是否可连接。
6. 后端日志中是否有错误。
7. Resource 是否写入 `error_message`。

------

### 32.4 问答没有引用来源

检查：

1. 课程资源是否 ready。
2. chunks 表是否有切片。
3. ChromaDB 是否有向量。
4. Retriever 是否限定了正确 course_id。
5. 相似度阈值是否过高。
6. Reranker 是否失败。
7. sources 格式是否被前端正确解析。

------

### 32.5 Agent 工作流返回占位内容

检查：

1. 是否调用了 Mock Workflow。
2. `AGENT_MOCK_WORKFLOWS_ALLOWED` 是否为 false。
3. Workflow LLM 节点是否接入真实 LLM Client。
4. Service 主链路是否错误切换到未完成 Agent。
5. Prompt Builder 是否正确注入模板。

------

### 32.6 MCP Tool 调用失败

检查：

1. `MCP_ENABLED` 是否为 true。
2. MCP Server 是否启用。
3. Tool 是否注册。
4. 当前用户角色是否有权限。
5. course_id 是否属于当前用户。
6. Tool 参数是否符合 Schema。
7. 是否超过超时限制。
8. 是否写入 `mcp_tool_calls` 错误记录。

------

### 32.7 Skill 执行失败

检查：

1. `SKILLS_ENABLED` 是否为 true。
2. Skill 是否注册。
3. Skill 是否启用。
4. 当前用户角色是否允许使用。
5. 输入参数是否符合 Schema。
6. Skill 依赖的 MCP Tool 是否可用。
7. LLM 是否配置正确。
8. 是否写入 `skill_runs` 错误记录。

## 35. 本文档总结

EduAgent 环境配置需要支持：

```text
基础课程系统
```

```text
4. 保证 JWT、CORS、存储配置安全
5. 新增 Agent / MCP / Skills 配置

7. 开启 Guardrails 和执行审计
8. 清理 .env、api-key.txt、venv、node_modules、上传文件和模型缓存
```
