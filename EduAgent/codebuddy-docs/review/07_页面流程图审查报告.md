## 1. 审查目的

本文档用于审查 EduAgent 项目前端页面流程、路由结构、权限控制、API 对接关系、页面状态设计，以及新版课程智能体平台所需的页面扩展是否合理。

新版 EduAgent 已经从普通课程 RAG 问答系统升级为：

```text

+ MCP 工具生态
+ Skills 技能系统
+ Agent Orchestrator
+ 多智能体协作
```


## 3. 总体审查结论

### 3.1 总体结论

当前 EduAgent 前端页面体系已经具备基础课程系统能力，包括：

1. 登录注册。
2. 首页。
3. 课程列表。
4. 课程资源。
5. 智能问答。
6. 教学任务。
7. 教学报告。
8. 课程设置。
9. 个人中心。
10. 管理员用户管理。
11. 管理员系统设置。

这些页面能够支撑基础业务闭环：

```text
教师创建课程
→ 学生加入课程
→ 教师上传资源
→ 学生问答
→ 教师生成任务
→ 教师生成报告
```

但如果项目目标是“课程智能体平台”，当前页面还存在明显不足：

```text
缺少资源分析页面
缺少教学设计页面
缺少学习路径页面
缺少代码辅导页面
缺少课程技能页面
缺少 Agent 执行记录页面
缺少 Skills 管理页面
缺少 MCP 管理页面
缺少 MCP Tool 调用记录页面
缺少 Skill 执行记录页面
```

因此，当前页面体系只能支撑“课程 RAG 应用”，还不能完整支撑“MCP + Skills + 多智能体协作”的平台定位。

问题是：


### 5.2 修复建议


1. 用户点击课程卡片后进入资源页。
2. 用户直接访问 `/courses/:courseId` 不出现空白页。
3. 页面仍能正确加载课程详情。
4. 课程侧边栏仍正常显示。

## 7. 前后端 API 一致性审查

### 7.1 P0：批量上传路径不一致

当前风险：

```text
前端可能使用：/resources/batch-upload
后端实际接口：/resources/upload-batch
```

影响：

1. 批量上传失败。
2. 页面显示上传异常。
3. 教师无法批量导入课程资料。

修复建议：

```text
统一前端路径为：
POST /api/v1/courses/{course_id}/resources/upload-batch
```

涉及文件：

```text
frontend/src/api/resources.ts
frontend/src/views/CourseResourcesView.vue
```

验收标准：

1. 教师可批量上传多个文件。
2. 后端返回每个资源的上传结果。
3. 上传后资源状态进入异步处理流程。
4. 前端轮询状态正常。

------

### 7.2 P0：任务生成字段不一致

当前风险：

```text
前端字段：extra_instructions
后端字段：additional_instructions
```

影响：

1. 教师填写的额外要求无法传给后端。
2. 生成任务不符合教师补充要求。
3. 前后端类型不一致。

修复建议：

```text
统一为 additional_instructions
```

涉及文件：

```text
frontend/src/api/tasks.ts
frontend/src/types/index.ts
frontend/src/views/CourseTasksView.vue
backend/app/schemas/task.py
```

验收标准：

1. 教师填写额外要求后，后端能收到该字段。
2. LLM 生成任务时能使用该字段。
3. TypeScript 类型和 Pydantic Schema 一致。

------

### 7.3 P0：Token 刷新链路不完整

当前风险：

```text
前端自动刷新依赖 refresh_token，
但后端登录响应可能未返回 refresh_token。
```

影响：

1. access_token 过期后用户被迫重新登录。
2. 前端 401 自动刷新逻辑失效。
3. 用户体验较差。

修复建议：

1. 后端登录接口返回 `refresh_token`。
2. 前端登录后保存 `refresh_token`。
3. 401 时调用 `/auth/refresh`。
4. refresh 失败后退出登录。

涉及文件：

```text
backend/app/api/auth.py
backend/app/schemas/user.py
frontend/src/api/client.ts
frontend/src/stores/auth.ts
```

验收标准：

1. 登录响应包含 access_token 和 refresh_token。
2. access_token 过期后自动刷新成功。
3. refresh_token 失效后自动退出登录。

------

### 7.4 P1：报告删除按钮与后端接口不一致

当前风险：

```text
前端可能存在删除报告按钮，
但后端可能没有 DELETE /reports/{report_id} 接口。
```

影响：

1. 用户点击删除后失败。
2. 页面功能误导。
3. 前后端契约不一致。

修复方案二选一：

方案 A：移除前端删除按钮。

方案 B：补充后端接口：

```http
DELETE /api/v1/courses/{course_id}/reports/{report_id}
```

建议：

```text
如果当前阶段没有明确删除报告需求，优先移除按钮。
```

------

### 7.5 P1：管理员更新用户参数格式不一致

当前风险：

```text
前端 updateUser 使用 JSON Body，
后端可能接收 Query 参数。
```

影响：

1. 管理员修改用户角色或状态失败。
2. 后台用户管理不稳定。

修复建议：

```text
统一为 PATCH /admin/users/{user_id}
请求体使用 JSON Body。
```

涉及文件：

```text
frontend/src/api/admin.ts
frontend/src/views/admin/AdminUsersView.vue
backend/app/api/admin.py
backend/app/schemas/user.py
```

## 16. CodeBuddy 页面开发要求

CodeBuddy 修改页面时必须遵守：

1. 不得重建前端项目。
2. 不得替换 Vue 3 技术栈。
3. 不得引入未确认的大型 UI 框架。
4. 新 API 必须写入 `frontend/src/api/`。
5. 新类型必须写入 `frontend/src/types/`。
6. 新页面必须写入 `frontend/src/views/`。
7. 新路由必须写入 `frontend/src/router/index.ts`。
8. 新页面必须配置权限。
9. 新菜单必须按角色显示。
10. AI Markdown 内容必须使用 MarkdownRenderer。
11. 所有删除操作必须二次确认。
12. 所有页面必须处理 loading、empty、error、normal 状态。
13. 不得在前端保存模型 API Key。
14. 不得展示完整系统 Prompt。
15. 不得展示 MCP Tool 敏感参数。
16. 不得展示服务器真实文件路径。
17. 新页面必须与 `04_API接口文档.md` 保持一致。
18. 新页面必须与 `07_页面流程图.md` 保持一致。

## 17. 审查总结

当前 EduAgent 前端已经能够支撑基础课程系统和 RAG 问答主流程，但还不能完整支撑新版课程智能体平台。

当前页面能力：

```text
登录注册
课程管理
资源管理
智能问答
教学任务
教学报告
课程设置
用户管理
系统设置
```

新版智能体平台还需要补齐：

```text
课程资源分析
教学设计
学习路径
代码辅导
课程技能
课程 Agent 执行记录
Skills 管理
MCP 管理
MCP Tool 调用记录
Skill 执行记录
全局 Agent 执行记录
```

当前最优先修复：

```text
1. 批量上传路径不一致
2. 任务字段不一致
3. Token 刷新链路不完整
4. 课程根路由无默认重定向
5. 报告删除按钮与后端接口不一致
6. 管理员更新用户参数格式不一致
```

后续页面扩展顺序：

```text
1. 修复现有页面和 API 问题
2. 新增 Agent / Skill / MCP TypeScript 类型
3. 新增 Skills、MCP、Agent API 封装
4. 新增课程内智能体页面
5. 新增管理员智能体平台页面
6. 补充权限、状态、移动端和安全验收
```

完成以上改造后，EduAgent 前端才能真正体现：

```text
课程业务系统
+ RAG 问答
+ MCP 工具生态
+ Skills 技能系统
+ 多智能体协作
+ 执行审计可视化
```
